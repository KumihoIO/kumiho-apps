# macOS Distribution & Updates (Developer ID + Sparkle)

This project supports distributing outside the Mac App Store using:

- A **DMG** for first install
- **Sparkle** for in-app updates

## Why Sparkle

Sparkle is the de-facto standard updater for macOS apps distributed outside the App Store.
It provides a native update UI and secure update verification.

## Setup

### 1) Add Sparkle feed URL + public key

Set these in [macos/Runner/Configs/AppInfo.xcconfig](../macos/Runner/Configs/AppInfo.xcconfig):

- `SPARKLE_FEED_URL` (HTTPS URL to your `appcast.xml`)
- `SPARKLE_PUBLIC_ED25519_KEY` (public key generated by Sparkle)

The app reads these from Info.plist keys:

- `SUFeedURL`
- `SUPublicEDKey`

### 2) Generate Sparkle keys (one-time)

Install Sparkle’s command line tools (from Sparkle release), then run:

- `generate_keys`

Put the **public** key into `SPARKLE_PUBLIC_ED25519_KEY`.
Store the **private** key in CI secrets for signing releases.

### 3) Build artifacts

- `./scripts/macos/build_release.sh`

Outputs:

- `dist/macos/Kumiho Browser.dmg`
- `dist/macos/Kumiho Browser.zip` (for Sparkle)

### 4) Generate appcast + sign update

After uploading artifacts to your download host, run:

- `SPARKLE_ED25519_PRIVATE_KEY=/path/to/private.key ./scripts/macos/sparkle_appcast.sh --base-url https://downloads.example.com/kumiho-browser/`

Upload:

- `dist/macos/appcast.xml`
- `dist/macos/Kumiho Browser.zip`

## CI (tag-triggered publishing)

On tags matching `browser-v*`, the workflow builds artifacts and publishes them to:

- `git@github.com:kumihoclouds/kumiho-browser.git`

It expects these GitHub repo secrets:

- `KUMIHO_BROWSER_DEPLOY_KEY`: SSH private key with write access to `kumihoclouds/kumiho-browser`
- `SPARKLE_ED25519_PRIVATE_KEY`: Sparkle Ed25519 private key text (used for signing the update ZIP)

## In-app UI

A native menu item exists:

- **Kumiho Browser → Check for Updates…**

This calls Sparkle’s updater (`SPUStandardUpdaterController`).

## Signing & Notarization

For production distribution, you should:

- Code sign with **Developer ID Application**
- Notarize with Apple Notary service

The build script supports optional signing if you set:

- `CODESIGN_IDENTITY="Developer ID Application: … (TEAMID)"`

Notarization is best handled in CI (xcrun notarytool + stapler).
